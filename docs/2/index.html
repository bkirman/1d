2. Editing and running code on the Plasma

Now let's look at how we will be editing the code. Let's just make a light flash on and off.

With Thonny open, look at the left side of the screen. The top part is the file system of your computer. This is handy if you are programming on your computer and copying it over. However for this guide, we will code directly onto the Plasma device. The bottom box should look something like this:

Double click the main.py here, and it will open that file in the main Thonny window.

If you haven't done much programming before, don't panic. We're here to help, and you can't break the Plasma. If something goes wrong, ask for help. We can always give you a new copy of this file.

This file is the code that runs on the Plasma when it is powered up. The grey lines that start with # are comments, and the Plasma does not run these lines, so we've used them to tell you a bit more about what is going on.

In the menu at the top of the Thonny window there are a few key buttons to know about. The save icon is important because we must manually save our changes to the Plasma.

The next button is the green play button. Press that one now.

If it isn't already, the light on the Plasma should turn green. The plasma is running this code. But there isn't anything to see yet. Let's make it interactive.

Look at the Plasma board, next to the green bit of circuitboard there are 3 buttons - RESET, BOOT, and A. If your code ever gets stuck, RESET is like switching it off and on again. Try it now if you like. You might see an error message but it should start running again.

The BOOT button is forbidden knowledge.

A, is a button we can use as an input into our games. Press it now!

Nothing happens :( we need to tell the Plasma to "listen" for the buttons and to do something with it.

Press the 🛑 stop button in Thonny to stop the Plasma from running the code. It might give you an error but don't worry, it's just complaining because we stopped it. We must stop the Plasma to edit the code.

Back in the code, look for line 24. It is below a comment that says # Connect the controller on the line below

Let's follow that instruction. Click on the empty white area below that, on line 24, and type the following:

     engine.setButtons(buttons)

We're using a language called Python, which is easy to read but has a very strict rule about indentation. Code must always be tidily aligned otherwise you will get errors and bugs.

So, you should be sure there are 4 space characters at the beginning of the line you just wrote. The first e you wrote on line 24 should be directly below the # on line 23.

Once you have done, press play.

The light turns green again. But now see what happens when you press the A button on the Plasma.

Congrats! You have made our first game: Pressy Pressy Lighty Lighty!

Have a read through the main.py file, to see what is happening, but don't change anything else for now. We'll get to that later.

Things that might have gone wrong

An IndentationError: unexpected indent error means that the code isn't aligned properly. Everything up to and including def main(): should have no spaces before it, and everything afterwards should have 4 spaces at the front. (until line 50)

An error like device is busy or back end not ready means that code is already running. Press stop 🛑 in Thonny and try again.

Advanced

The Plasma is programmed using Micropython. This is like Python but micro-er. If you've programmed in Python you will be very familiar, but note that Micropython does not have the full power and libraries available. The Plasma only has 4mb of memory!

The reference guide for MicroPython details what language features are available.

You can also program the Pico using Circuitpython, or even C. But the workshop is written for Micropython, and it can do everything we need.
