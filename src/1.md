# {{ title }}

You will need a computer to be able to program our mini computer. We will be using Thonny, which is a simple free and open source editor that is perfect for our purposes.

Download and install the [Thonny IDE](https://thonny.org/) (Linux/Mac/Win)

Once it is installed, run it.

In the kit, find the Plasma 2350W in the black antistatic bag - this will be the brain of the system. Remove it from the package and take a look at it - this thing is cool - it's a whole computer! For the rest of the guide we'll just call it the "Plasma".

<img src="/images/plasma_board.png" alt="A photograph of the Plasma 2350W board from Pimoroni, showing components mounted on a tiny pcb"/>

Plug in the Plasma to your computer, using the USB cable. The Plasma should light up an LED next to the rainbow icon 🌈.

Now we need to tell Thonny to connect. We do this at the bottom right of the Thonny Window. It might say ``"Local Python 3..."`` or similar. Click this text and a menu will pop up. Select ``MicroPython (Raspberry Pi Pico) Board in FS Mode``.

<img src="/images/thonny.png" alt="A screenshot of Thonny, and the MicroPython option selected in the menu"/>

You should now see a list of files on the left hand side. You should also see something in the "Shell" window at the bottom. Something like ``"MicroPython cyw43-breakout/1.27.0,   on 2026-03-02; Pimoroni Plasma 2350 (LTE + WiFi) with RP2350"``. This is the Plasma telling us about itself. Below that is three chevrons: ``>>>``. Click here any type ``print("Hello EMF")`` and hit return. It should then say "Hello EMF". That code was running on your Plasma! 

This is the main way we are going to speak to our Plasma. The next step is to investigate the code a bit.

### Plasma v Pico?
You might already notice there is a bit of mixed language here. Sometimes I'll call it Pico, sometimes Plasma, and sometimes interfaces will too (i.e. Thonny!). We are technically working with the Pimoroni Plasma 2350W board, which uses the RP2350 chip from Raspberry Pi. Raspberry Pi have their own device called the ["Raspberry Pi Pico 2"](https://www.raspberrypi.com/products/raspberry-pi-pico-2/). Both are mostly compatible with each other, but Pimoroni did a great job making the Plasma easy to work with for LED strips in particular. It's perfect for us, where the Pico would require more wiring and code.

Since it is compatible, you can also run most Pi Pico examples on the Plasma, and follow tutorials (but you can't run most Plasma examples on the Pico, because of the custom LED features).

If you do follow external tutorials, just be sure to use the [pinout guide from Pimoroni](https://shop.pimoroni.com/products/plasma-2350-w) if you need to add components.

## Things that might have gone wrong

If Thonny does not recognise the Plasma, check the lights on the Plasma - is it green? Try pressing the RESET button. Did that work?
Try double checking the menu at the bottom right of Thonny.

If it is still not recognised, try using a different USB cable or USB port on your computer. 

## Advanced
We are using Thonny because it is clean and simple to use, but you might prefer working in a tool like VS Code. [Raspberry Pi have made an official plugin](https://marketplace.visualstudio.com/items?itemName=raspberry-pi.raspberry-pi-pico) to support it. This is how I work because it can also integrate with Git or other VCSs.
